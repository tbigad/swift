Разделил на три схемы, одну для отладки, вторую для разработки, третью для релиза и выпуска приложения. Сделал это для быстрого и удобного перехода между схемами. 
В NoteRelease: установил Build Configuration в Release, убрал опцию Main Thread Checker. В релизе нам нужна скорость а не проверки.
Убрал любой дебаг логгин приложения, например из функции print() послердством установки переменной OS_ACTIVITY_MODE в disable. Логи в релизе никчему. 

В NoteDevelop остались стандартные настройки. Это основная схема для разработки. 

В NoteDebug: Включил все доступные опции для отслеживания утечек памяти. Для того он дебаг и нужен что бы побольше паранои. 

Настройки таргета: "General" Добавил Display name, наверное он очень нужен.
